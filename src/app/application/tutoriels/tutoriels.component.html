Mini tutoriel Angular

Angular est un framework open source développé par Google. Écrit en JavaScript, il permet la création de Single Page Applications. Cela permet une meilleur fluidité puisque celle-ci s'ont accesible via une page web unique et évite de devoir charger la page à chaque action.

Pour utiliser Angular il faut d'abord l'installer avec la commande "npm install -g @angular/cli@10.0.0".

Pour ensuite installer et configurer le serveur Json on doit faire les commandes suivantes: "npm i g json-server" et "npm i faker".

Optionnellement vous pouvez installer le compilateur Typescript pour travailler à partir d'un projet Angular. Les commandes pour l'installation sont: "npm install -g typescript@latest" et "npm install -g ts-node".

Mini tutoriel CRUD

L'acronyme CRUD signifie Create, Read, Update et Delete. Le CRUD est utilisé pour créer et gérer les données dans la base de données (INSERT, SELECT, UPDATE, DELETE).

Mini tutoriel TypeScript

TypeScript est un language qui à été développé en 2012 par Microsoft. Ce language open source est un sur-ensemble de Javascript, ce qui veux dire que tout code valide en Javascript est également valide en TypeScript. Ce language permet la programation orientée objet et le typage. Pour le compiler il faut utiliser l'outil de compilation TypeScript pour le transpiler en Javascript. Ce qui signifie que le code exécuté sera un équivalent Javascript du code TypeScript compilé.

Mini tutoriel des relations

OneToOne

Dans une relation OneToOne, un élément dans une table est associé avec un et seulement un élément d'une autre table.

Exemple: Il a une relation OneToOne entre les tables permisDeConduire et personne. Cela veux dire qu'un permis de conduire doit absolument avoir une personne. Dans ce cas numAssSocial est la clé primaire de la table personne et est une clé étrangère dans la table permisDeConduire.

OneToMany

Dans une relation OneToMany, un élément dans une table est associé avec un ou plusieurs éléments d'une autre table.

Exemple: Il a une relation OneToMany entre les tables commande et client. Cela veux dire qu'un client peut avoir une ou plusieurs commandes. Une commande tant qu'a elle doit avoir absolument un client. Dans ce cas numClient est la clé primaire de la table client et est une clé étrangère dans la table commande.

numClient (PK)

commande
numCommande (PK)
numClient (FK)

ManyToMany

Dans une relation ManyToMany, plusieurs éléments d'une table sont associer avec plusieurs éléments d'une autre table.

Exemple: Il a une relation ManyToMany entre les tables produit et client. Cela veux dire qu'un client peut acheter plusieurs produits et un produit peut être acheté par plusieurs clients. Lorsqu'il a une relation ManyToMany il faut la séparer en deux relation OneToMany et rajouter une autre table entre les 2.

Héritage

L'héritage est un mécanisme qui permet a une classe d'hériter des caractéristiques de la classe parents.

Exemple: Il a une relation entre lea table animal et chien. Cela veux dire qu'un chienva hériter des caractéristique de la table animal.

Composition / Agrégation

La composition permet a un objet d'être plus complexe en l'associant à d'autres objets

L'agrégation permet de définir une classe comme étant liée à plusieurs entités de classes diffentes. Cela n'entraîne pas l'appartenance.
